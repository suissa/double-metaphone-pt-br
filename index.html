Fonética pt-BR + busca híbrida

Como usar códigos fonéticos para cobrir mais cenários de busca

Esta página explica, de forma prática, onde a fonética entra em uma busca por similaridade semântica. Ela não “adivinha” sozinha a palavra correta: ela cria uma chave de som para recuperar candidatos que embeddings, busca textual ou distância de edição podem deixar escapar.

Resposta curta

Sim, você pode usar a fonética para transformar uma palavra escrita errada em candidatas prováveis de palavra correta.

O fluxo recomendado é: gerar código fonético da consulta, buscar palavras/documentos com código parecido, combinar com busca textual e semântica, e só então ranquear os melhores resultados.

Por que aqueles testes existem?

Os testes abaixo não tentam dizer que essas palavras são iguais. Eles documentam decisões de pronúncia úteis para busca. Cada expectativa mostra como uma palavra vira um par de códigos: um primário e um secundário.

Teste O que está sendo coberto Como isso ajuda na busca
m('Brasil') => ['PRZU', 'PRSL'] B aproxima de P, s entre vogais pode soar como z, e l final pode soar como u no Brasil. Ajuda a aproximar variações de fala, sotaque e transcrição como “Braz...”/“Bras...” ou final “...il”/“...iu”.
m('bom') => ['P', 'P'] B converge para P e m final após vogal marca nasalização, sem criar uma consoante forte no fim. Evita que “bom”, “bon” e transcrições nasais próximas fiquem excessivamente distantes.
m('hífen') => ['AF', 'AF'] H inicial é mudo, acento é normalizado, e n final após vogal é tratado como nasalização. Permite recuperar termos mesmo quando o usuário escreve sem acento ou quando o áudio não preserva o h gráfico.
m('show') => ['XU', 'XV'] sh é aproximado para som de x e w final pode receber leitura vocálica/semivocálica. Cobre estrangeirismos comuns em português brasileiro, como “show”, que usuários podem transcrever pelo som.
m('yamaha') => ['IM', 'IM'] y próximo de vogal vira som de i e h interno é ignorado. Ajuda nomes de marca ou nomes próprios estrangeiros que entram em uma busca em português.
m('toy') => ['TI', 'TI'] y final depois de vogal recebe leitura de i. Ajuda quando um termo estrangeiro curto aparece em catálogo, produto ou transcrição de fala.

Onde a fonética entra em uma busca semântica híbrida?

Em uma busca híbrida, você normalmente combina sinais diferentes. A fonética é mais um sinal, focado em erro de escrita, som parecido, transcrição de áudio e troca de consoantes. Ela complementa, mas não substitui, embeddings ou busca textual.

1. Léxico

BM25 / texto exato

Encontra documentos que contêm a palavra ou tokens próximos à consulta. É forte para nomes, códigos e termos exatos.

2. Semântico

Embeddings

Encontra conteúdo com significado parecido, mesmo usando palavras diferentes, como “remédio” e “medicamento”.

3. Fonético

Código de som

Encontra candidatos quando o usuário escreveu pelo som, como xave para chave ou ezame para exame.

Exemplo de pipeline

Imagine que sua base tem a palavra correta exame, mas o usuário digitou ezame ou o áudio foi transcrito dessa forma. 

import {doubleMetaphone} from 'double-metaphone'

const query = 'ezame'
const queryCodes = doubleMetaphone(query) // ['AZM', 'AZM']

// No índice, a palavra correta também guarda códigos fonéticos.
const indexedTerm = {
  text: 'exame',
  phonetic: doubleMetaphone('exame') // ['AZM', 'ASM']
}

const phoneticMatch = indexedTerm.phonetic.some(function (code) {
  return queryCodes.includes(code)
})

console.log(phoneticMatch) // true

Depois desse match fonético, você não precisa aceitar automaticamente que ezame significa exame. O ideal é usar o match para adicionar candidatos ao ranking final.

Como ranquear os candidatos

Uma forma simples é combinar pontuações. Os pesos dependem do seu domínio, mas a ideia é esta: 

scoreFinal =
  0.45 * scoreSemantico +
  0.30 * scoreTextual +
  0.15 * scoreFonetico +
  0.10 * scoreEdicao
Sinal Quando é forte Exemplo
0.45 Semântico Quando o significado é parecido. “atestado médico” encontra “documento de saúde”.
0.30 Textual Quando a palavra aparece de forma literal. “boleto vencido” encontra documentos com esses termos.
0.15 Fonético Quando a escrita está errada, mas o som é parecido. “xave” encontra “chave”; “ezame” encontra “exame”.
0.10 Edição Quando há poucas inserções, remoções ou trocas de letras. “relatorio” encontra “relatório”.

Isso converte palavra errada em correta?

Não diretamente. O algoritmo fonético não é um corretor ortográfico e não conhece o vocabulário da sua aplicação. Ele responde uma pergunta mais simples: “essa escrita soa parecida com outra?”.

Para converter uma palavra errada em uma correta, use este fluxo:

  1. monte um dicionário com termos válidos do seu domínio;
  2. salve os códigos fonéticos de cada termo;
  3. gere os códigos fonéticos da consulta;
  4. busque candidatos que compartilhem algum código;
  5. ranqueie por fonética, distância de edição, frequência e contexto;
  6. retorne a melhor sugestão ou mostre uma lista de alternativas. 
// Consulta digitada ou vinda de áudio:
'xave'

// Candidatos por fonética:
['chave', 'chaves', 'enxave']

// Ranking final escolhe:
'chave'

Regra prática

Use fonética para aumentar recall: trazer mais candidatos que talvez estejam escritos errado. Use semântica, texto e distância de edição para aumentar precisão: escolher quais desses candidatos realmente fazem sentido para o usuário.
